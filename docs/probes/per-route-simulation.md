## Interpretation

If the per-route simulation shows pages averaging much less than 65 KB, then **per-route bundle splitting is a prerequisite to T5-B/C** — it alone delivers a multi-fold reduction with no other change to the framework.

Note: this simulation overestimates the win slightly because the layout chrome (sidebar, search palette, ToC) IS shared across pages. A production per-route splitter would emit:
- 1 shared chunk for layout + framework runtime
- N small per-page chunks (just the page's own view code)

The shared chunk would be loaded once and cached; each navigation only fetches the small per-page chunk. So actual gains are larger than this simulation's average suggests (the layout is amortized).
